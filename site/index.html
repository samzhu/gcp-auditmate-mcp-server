macOS 版本 (Apple Silicon/ARM 晶片)

適用於 M1、M2、M3 等 Apple Silicon 晶片的 Mac 電腦。

下載連結：直接下載 或 ZIP 壓縮包

軟件物料清單 (SBOM)

我們提供本應用程式的軟件物料清單 (SBOM)，列出所有依賴組件及其版本資訊，有助於了解軟件組成、識別潛在安全風險及遵循開源許可證要求。

可以通過以下格式獲取最新版本的 SBOM：

  • JSON 格式 - 適合機器讀取和進一步處理
  • XML 格式 - 與多種 SBOM 工具相容
  • HTML 報告 - 提供直觀的視覺化界面，展示依賴關係、許可證分佈和潛在漏洞

提示：HTML 報告由 CycloneDX Sunshine 工具生成，包含互動式的圖表和表格，是了解軟件依賴結構的最佳方式。

使用說明

步驟 1: 下載並解壓執行檔

下載適合您 Mac 系統晶片類型的執行檔到您習慣的目錄，例如 /Users/username

如果您不確定您的 Mac 使用哪種晶片，請點擊左上角蘋果圖示 > 關於這台 Mac 查看處理器資訊。

步驟 2: 設定執行權限和移除安全限制

由於 macOS 的安全機制，您需要執行以下兩個命令來設定執行權限並移除安全屬性（路徑請替換為您的實際路徑）：

Intel Mac 用戶：

chmod +x /Users/username/gcp-auditmate-mcp-server-darwin-x86_64
xattr -cr /Users/username/gcp-auditmate-mcp-server-darwin-x86_64

Apple Silicon/ARM Mac 用戶：

chmod +x /Users/username/gcp-auditmate-mcp-server-darwin-aarch64
xattr -cr /Users/username/gcp-auditmate-mcp-server-darwin-aarch64

指令說明：

  • chmod +x：賦予檔案執行權限，允許檔案作為程式運行
  • xattr -cr：移除 macOS 對從網路下載檔案施加的隔離屬性和安全限制，解決「來自未識別的開發者」的警告問題

步驟 3: 登入 Google Cloud

在終端機執行以下命令：

gcloud auth application-default login

步驟 4: 設定 Claude Desktop

打開 Claude Desktop 設定檔。在 macOS 上路徑通常為：/Users/username/Library/Application Support/Claude/claude_desktop_config.json

新增以下設定，請將命令路徑替換為您實際的執行檔路徑：

{
    "mcpServers": {
        "gcp-auditmate-mcp-server": {
            "command": "/Users/username/gcp-auditmate-mcp-server-darwin-x86_64"
        }
    }
}

或者對於 ARM 版本：

{
    "mcpServers": {
        "gcp-auditmate-mcp-server": {
            "command": "/Users/username/gcp-auditmate-mcp-server-darwin-aarch64"
        }
    }
}

步驟 5: 使用 Claude Desktop

打開 Claude Desktop，您會看到 MCP tool 已載入。

點擊 MCP tool，您將看到 gcp-auditmate-mcp-server 的功能資訊。

接著請他幫您進行 GCP 資源的自我查核。
